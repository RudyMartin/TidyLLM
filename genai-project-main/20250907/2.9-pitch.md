

# 🔑 How to Support Tables & Charts in **Custom Checks**

### Extend the CustomChecks file schema:

| section\_id | custom\_question                         | evidence\_display | output\_format |
| ----------- | ---------------------------------------- | ----------------- | -------------- |
| Findings    | Summarize number of findings by severity | summary           | table          |
| Testing     | Show accuracy trend over iterations      | span+page         | chart          |
| Governance  | List reviewers and their roles           | span              | table          |

# 🛡 Guardrails

* **Allowed output\_format values:** `text` (default), `table`, `chart`.
* **Chart types limited to:** `line`, `bar`, `pie`. (Keep it simple.)
* **System rule:** Every table/chart must cite evidence (page refs or counts) → prevents “pretty pictures with no grounding.”
* **Audit safety:** Store the *raw data* in JSON even if the client only sees the chart in the PDF.

# 🎯 Pitch to Client

> “Along with Custom Checks, you can decide if the output should be **plain text, a table, or a chart**.
>
> * Tables: great for counts and comparisons.
> * Charts: line, bar, or pie for quick visuals.
> * Text: default for commentary.
>
> The system will still log the raw evidence for audit, but your report will show it in the format you choose.”
