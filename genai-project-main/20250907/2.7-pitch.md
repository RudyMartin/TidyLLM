# 📑 Report Structure

### **Official Checklist** (locked, compliance-driven)

* Always the Y/N items tied to standards.
* Scored, traceable, evidence-backed.
* Output: CSV + PDF section.

### **Custom Checks** (client’s playground)

* Whatever extra questions they add/edit.
* Commentary only → does **not** affect official score.
* Output: In a separate section of the PDF/Word + optional JSON export.

**Official Checklist**

| Section      | Question                   | Answer  | Reason / Evidence                     |
| ------------ | -------------------------- | ------- | ------------------------------------- |

# 🛠 Input Format

They keep **two tabs** in Excel (or two sheets in the JSON schema):

1. **Checklist** (locked, official)

   * `question_id`, `section_id`, `question_text`, `expected_type=bool`
2. **Custom Checks** (editable by them)

   * `custom_id`, `section_id`, `question_text`, `expected_type=commentary`

---

# 🎯 How to Frame This to the Client

> “You’ll always have the **Checklist** for compliance.
> If you want to experiment, add your own **Custom Checks** in the extra tab. Those show up in the report under a separate section called *Custom Checks*. They won’t change the official score, but they’ll give you commentary where you want more depth.”
