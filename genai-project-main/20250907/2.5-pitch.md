# 🎛 Two-Layer Setup (Control vs. Safety)

## 1. **Stable Core (non-editable)**

* The official **QA Checklist** (Y/N questions tied to standards: SR 11-7, OCC 2011-12, internal policy).
* This is **locked** for governance and compliance.
* Always runs in the background → guarantees coverage.

## 2. **Editable Guidance (playground layer)**

* A separate **“Guidance Questions” tab** (Excel, JSON, or web form).
* Clients can add / change **leading questions, commentary style, or extra clarifiers**.
* These **do not replace the core checklist**; they **add extra commentary in the report**.
* Outputs show them clearly in a separate section:

  * “Core QA Answers” (official)
  * “Custom Guidance Commentary” (playground)

---

# 📄 Report Example

**Section: Data Quality**

* **Checklist Answer:** Y
* **Reason:** Validation steps documented on page 12.
* **Evidence:** “Data validation performed” (p. 12).

**Custom Guidance Commentary (client-authored):**

* Q: *“Does the validation cover external vendors?”*
* A: Found no mention; mark as pending.

---

# 🛡 Guardrails (to keep them from breaking it)

* **Schema lock:** Every guidance question must have:

  ```json
  {
    "question_id": "GQ-101",
    "section_id": "DataQuality",
    "question_text": "...",
    "expected_type": "commentary"
  }
  ```
* **No overwrite:** They can only create “extra questions,” not delete core ones.
* **Separate scoring:** Custom guidance never changes the official Y/N/scorecard — it just adds extra narrative.

---

# 🎯 How to Sell It Back to Them

> “Yes, you can change or add guidance questions anytime — think of it as your **custom lens**. The core checklist (the compliance must-haves) is always locked, so you never lose coverage. Your guidance will show up in its own section of the report as commentary.”

That way, they get their “prompt-engineer” playground, but you keep the core QA safe and consistent.
