# 🛠️ How-to: Run Project Diagnostics

Use this script to verify the health and hygiene of your codebase. It checks for bad filenames, versioning issues, empty folders, and missing `__init__.py` files. Ideal for pre-release QA, commit hooks, or CI/CD prechecks.

---

## ✅ What It Checks

| Check Type                  | Description                                                          |
|----------------------------|----------------------------------------------------------------------|
| 🚫 Banned Filenames         | Flags files like `z_*.py`, `file_index.py`, `admin_*.py`, etc.       |
| ⚠️ Unversioned Index Files  | Warns if `file_index.csv` exists without a `_YYYYMMDD` version        |
| 📂 Suspicious Folders       | Flags empty folders that aren't `.git` or `__pycache__`               |
| 📦 Missing `__init__.py`    | Detects folders with `.py` files that are missing `__init__.py`       |

---

## 🚀 How to Run

From project root or the `admin/` folder:

```bash
python admin/project_diagnostics.py
```

This will print a summary report directly to your console.

---

## 🔧 Auto-Fix Mode: Create Missing `__init__.py`

To automatically create `__init__.py` files in folders that need them:

```bash
python admin/project_diagnostics.py --fix-init
```

Each missing file will be created with this content:

```python
# Auto-generated by diagnostics tool
```
