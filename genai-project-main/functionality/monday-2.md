Responsive behaviour
Mobile (< 768 px) – Keep the hamburger menu to collapse navigation and a bottom tab bar for quick access to the new top‑level pages (Dashboard, Documents, Tasks, Projects, Compliance, Automation, Reports, Admin, Help). Continue using single‑column layouts with stacked cards and large, touch‑optimised buttons.

Tablet (768 – 1279 px) – Use a collapsible sidebar with overlay behaviour for the left navigation. With fewer main pages, the sidebar will be easier to scan. Preserve two‑column card layouts where appropriate and ensure content condenses gracefully.

Desktop (≥ 1280 px) – Retain the full horizontal tab navigation for the nine pages, while keeping the sidebar visible with expand/collapse options. Use multi‑column layouts and rich data tables for reports, documents, admin screens, etc.

Applying the design system
Colour palette – Continue using Primary Blue (#2563eb) for key actions and navigation highlights across all pages. Use the existing Success Green, Warning Yellow, Error Red and Purple hues for status indicators, alerts and automation‑related features. Leverage the grey scale for backgrounds, borders and secondary text to maintain hierarchy.

Typography – Maintain a clear hierarchy with bold headers and readable body text. Monospaced fonts should still be used for numerical data or structured content (e.g. reports and audit trails).

Components – Keep the same card, table, form and button styles: cards with consistent padding, shadows and rounded corners; tables with alternating row shading and clear headers; forms with clear labels, validation states and hints; and buttons with distinct primary/secondary states. Status indicators (badges, progress bars) should remain colour‑coded based on the established palette.
