

| **Design Principle**        | **Appian Behavior**                                                                      |
| --------------------------- | ---------------------------------------------------------------------------------------- |
| Task-First                  | Pages focus on **“what do you want to do?”** not “what components do we have?”           |
| Minimal Clutter             | Users land on a clean layout with **clear next actions** (like buttons or cards)         |
| Context-Aware Subnavigation | When you click into a section, **sub-tabs or segmented views appear**, not long lists    |
| Uniform Page Types          | Appian loves **Records**, **Reports**, and **Actions** — they keep interfaces consistent |
| Guided Interactions         | Every page feels like a small, guided workflow (“Upload → Tag → Submit”)                 |
