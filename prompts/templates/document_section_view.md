# Document Section View Workflow Template
*Interactive document navigation and section-by-section analysis*

## Document Structure Analysis
```
Document Structure Breakdown:

Analyze the document and create a navigable structure for interactive browsing:

Document: {document_content}

Structure Analysis:
1. **Document Metadata**:
   - Title and document type
   - Reference numbers, case IDs, revision numbers
   - Creation/modification dates
   - Version information
   - Author/organization information

2. **Content Hierarchy**:
   - Main sections and subsections
   - Table of contents generation
   - Section numbering and organization
   - Cross-references and dependencies

3. **Section Classification**:
   - Header/summary information
   - Primary content areas
   - Data tables and structured content
   - Conclusions and recommendations
   - Appendices and supporting materials

4. **Navigation Structure**:
   - Clickable section index
   - Section relationships and flow
   - Important subsection highlights

Output as structured JSON for interactive navigation interface.
```

## Section Content Summarization
```
Section-by-Section Content Summary:

For each identified section, create concise summaries for quick navigation:

Section Content: {section_content}
Section Context: Part of larger document analysis

Summary Components:
1. **Section Overview** (1-2 sentence purpose)
2. **Key Information** (3-5 most important points)
3. **Critical Data** (specific numbers, dates, names, amounts)
4. **Action Items** (if any tasks or requirements mentioned)
5. **Risk Indicators** (potential issues or red flags)
6. **Related Sections** (cross-references to other parts)
7. **Business Relevance** (why this section matters)

Format for quick scanning and decision-making navigation.
```

## Section Deep-Dive Analysis
```
Detailed Section Analysis:

Comprehensive analysis when user selects specific section for detailed review:

Target Section: {section_title}
Section Content: {section_content}
Document Context: {document_overview}

Deep Analysis:
1. **Content Breakdown**:
   - Detailed content analysis
   - Purpose within overall document
   - Key concepts and terminology
   - Data quality and completeness

2. **Data Validation**:
   - Information accuracy assessment
   - Consistency with other sections
   - Missing or incomplete data identification
   - Format and standard compliance

3. **Risk and Compliance**:
   - Regulatory compliance considerations
   - Risk factors identified
   - Red flags or concerns
   - Areas requiring attention

4. **Business Impact**:
   - Decision-making implications
   - Financial or operational impact
   - Stakeholder considerations
   - Follow-up requirements

Present with expandable details and clear action recommendations.
```

## Interactive Navigation Report
```
Interactive Document Navigation Report:

Create comprehensive navigation aid for complex document review:

Full Document: {document_content}

Navigation Report:
1. **Document Overview**:
   - Executive summary of entire document
   - Document type and business purpose
   - Overall risk assessment
   - Key stakeholders affected

2. **Priority Sections**:
   - Most critical sections to review first
   - High-risk areas requiring immediate attention
   - Sections with decision-making impact
   - Time-sensitive information

3. **Section Interdependencies**:
   - How sections relate to each other
   - Information flow and logical sequence
   - Contradictions or inconsistencies
   - Supporting documentation references

4. **Review Workflow**:
   - Recommended review sequence
   - Estimated time per section
   - Required expertise per section
   - Approval/sign-off requirements

Format as interactive guide for efficient document review process.
```
