# QA Control Workflow Template
*Quality Assurance review and validation workflow*

## QA Document Intake and Validation
```
QA Document Intake Assessment:

Perform initial QA validation of the submitted document:

Document for QA Review: {document_content}

Intake Validation:
1. **Document Completeness**:
   - Required sections present
   - Mandatory fields populated
   - Supporting documentation attached
   - Revision history and versioning

2. **Format and Structure**:
   - Follows organizational standards
   - Consistent formatting and layout
   - Proper numbering and references
   - Table of contents and navigation

3. **Content Quality**:
   - Information accuracy and clarity
   - Language and terminology consistency
   - Data validation and verification
   - Cross-reference integrity

4. **Compliance Readiness**:
   - Meets submission requirements
   - Regulatory format compliance
   - Audit trail completeness
   - Approval workflow status

Output QA intake scorecard with pass/fail recommendations.
```

## QA Process Validation
```
QA Process and Procedure Validation:

Validate document against established QA processes and procedures:

Document Content: {document_content}
QA Standards: [Reference organizational QA standards]

Process Validation:
1. **Procedure Compliance**:
   - Follows established workflows
   - Meets process requirements
   - Adheres to quality standards
   - Implements required controls

2. **Quality Metrics**:
   - Quantitative quality measures
   - Performance indicator assessment
   - Benchmark comparisons
   - Trend analysis and patterns

3. **Process Effectiveness**:
   - Objective achievement assessment
   - Efficiency and resource utilization
   - Error rates and defect analysis
   - Customer satisfaction indicators

4. **Continuous Improvement**:
   - Process improvement opportunities
   - Best practice recommendations
   - Lessons learned integration
   - Future enhancement suggestions

Provide detailed process validation report with improvement recommendations.
```

## QA Risk Assessment and Mitigation
```
QA Risk Assessment and Quality Assurance:

Identify and assess quality risks in the document and processes:

Document for Risk Assessment: {document_content}

QA Risk Analysis:
1. **Quality Risk Identification**:
   - Potential defect sources
   - Process vulnerability points
   - Human error susceptibility
   - System and technology risks

2. **Impact Assessment**:
   - Customer impact evaluation
   - Business process disruption
   - Regulatory compliance risks
   - Financial and operational consequences

3. **Risk Mitigation Strategies**:
   - Preventive quality measures
   - Detective controls and monitoring
   - Corrective action procedures
   - Risk transfer and acceptance

4. **Quality Assurance Plan**:
   - QA checkpoint establishment
   - Testing and validation protocols
   - Review and approval processes
   - Monitoring and measurement systems

Format as comprehensive QA risk management plan.
```

## QA Approval and Sign-off
```
QA Final Review and Approval Decision:

Comprehensive final QA review for approval decision:

Complete Document Package: {document_content}

Final QA Assessment:
1. **Overall Quality Rating**:
   - Composite quality score
   - Individual criteria ratings
   - Benchmark comparison
   - Historical performance context

2. **Approval Recommendation**:
   - Approve without conditions
   - Approve with minor corrections
   - Conditional approval with requirements
   - Reject with specific remediation needs

3. **Sign-off Requirements**:
   - Required approver identification
   - Approval sequence and dependencies
   - Documentation and audit trail
   - Escalation procedures if needed

4. **Post-Approval Actions**:
   - Implementation readiness check
   - Communication and distribution plan
   - Monitoring and feedback mechanisms
   - Lessons learned documentation

Provide executive QA summary with clear approval recommendation and rationale.
```
